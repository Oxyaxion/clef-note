# The Pragmatic Programmer

A foundational book on software craftsmanship. The second edition (2019) feels remarkably fresh.

## Key ideas

**DRY — Don't Repeat Yourself**Every piece of knowledge must have a single, unambiguous representation in a system. Duplication leads to inconsistency and maintenance pain.

**Tracer bullets**Build thin, end-to-end slices of functionality early. Like tracer rounds, they let you see immediately if you're on target — before committing to a full implementation.

**The broken window theory**Don't leave bad code in place. One messy function signals that nobody cares, and entropy accelerates.

**Orthogonality**Components should be independent. A change in one should not require changes in others.

## Favourite quotes

> "You can't write perfect software. Software is too complex. But you can write better software."

> "Fix the problem, not the blame."
